# 项目 Bug 修复和样式优化总结

## 1. Bug 修复

### 问题描述
- 在题目页面的题目列表中点击题目后进入题目详情页，但详情页中是空白的，没有显示任何内容，也没有调用获取题目详情接口

### 根本原因
**代理配置不完整**：
- `next.config.mjs` 中只配置了 `/api/:path*` 到 `http://127.0.0.1:48080/:path*` 的代理
- 但项目中同时使用了两种 API 路径：
  - `/api/` 路径用于部分API
  - `/app-api/` 路径用于题目相关的API（`/app-api/biz/question/get` 和 `/app-api/biz/question/page`）
- 缺少对 `/app-api/` 路径的代理配置，导致这些请求无法到达后端服务器

### 解决方案
在 `next.config.mjs` 中添加对 `/app-api/` 路径的代理配置：

```javascript
async rewrites() {
  return [
    {
      source: '/api/:path*',
      destination: 'http://127.0.0.1:48080/:path*',
    },
    {
      source: '/app-api/:path*',
      destination: 'http://127.0.0.1:48080/app-api/:path*',
    },
  ]
},
```

**重要**：代理配置修改后需要重新启动开发服务器才能生效。

## 3. 技术细节

### 代理配置说明
项目使用两套API路径：
- `/api/` 用于用户认证等基础API
- `/app-api/` 用于业务相关API（题目、标签等）

确保 `next.config.mjs` 中同时配置了两种路径的代理，并且后端服务运行在 `http://127.0.0.1:48080`。

### 颜色方案
- 主色调：蓝色 (`blue-600`, `blue-700`)
- 辅助色：紫色 (`purple-600`) 用于渐变
- 状态色：
  - 简单：绿色 (`green-500`, `green-600`)
  - 中等：黄色 (`yellow-500`, `yellow-600`)
  - 困难：红色 (`red-500`, `red-600`)

### 响应式设计
- 保持原有的响应式布局
- 改善移动端的显示效果
- 使用 Flexbox 和 Grid 布局
- **重要**：分页组件在所有屏幕尺寸下都固定在底部

### 交互效果
- 添加 `transition-all duration-200` 过渡动画
- 悬停状态的缩放和颜色变化
- 焦点状态的边框和阴影效果

### 暗色模式支持
- 所有新增样式都支持暗色模式
- 使用 `dark:` 前缀确保良好的暗色显示效果
